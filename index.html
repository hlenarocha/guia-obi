Introdução

Sobre a OBI

A Olimpíada Brasileira de Informática (OBI) é uma competição que exige raciocínio lógico, habilidades em programação e a capacidade de resolver problemas de forma eficiente.

Se você está se preparando para a prova, é importante estar ciente das linguagens de programação permitidas, suas características e como utilizá-las adequadamente na prova.

Sobre o site

Este site, desenvolvido para o Projeto de Extensão "Estrutura de Dados, Algoritmos e Desafios de Programação" da Universidade Estadual de Ponta Grossa (UEPG), ajudará você a entender quais linguagens podem ser usadas na Modalidade Programação e algumas dicas importantes para maximizar seu desempenho.

Linguagens Permitidas

Na Modalidade Programação, as linguagens permitidas são: C, C++, Java, Python e JavaScript. No momento da prova, você deve selecionar uma delas para desenvolver o código de resolução do problema apresentado. Confira abaixo uma visão geral e dicas sobre cada linguagem:

C

  • Extensão de arquivo: .c
  • Funções de entrada/saída: scanf, getchar, printf, putchar
  • Dicas
    • Evite usar bibliotecas adicionais para entrada e saída, além das padrões.
    • C é uma linguagem eficiente, sendo uma boa escolha para tarefas que exigem alto desempenho.
    • Preste atenção ao gerenciamento de memória, pois é um fator crucial na correção.

C++

  • Extensão de arquivo: .cpp
  • Funções de entrada/saída: scanf, getchar, printf, putchar, ou os objetos cin e cout
  • Dicas
    • C++ oferece maior flexibilidade e recursos mais modernos que C, como a STL (Standard Template Library).
    • O uso de cin e cout facilita a manipulação de entrada/saída, mas scanf e printf são mais rápidos em grandes volumes de dados.
    • Utilize a STL para resolver problemas com estruturas de dados como vector, map e set para ganhar eficiência.

Java

  • Extensão de arquivo: .java
  • Funções de entrada/saída: Scanner, BufferedReader, BufferedWriter, System.out.println
  • Dicas
    • Lembre-se de que a classe principal deve ter o mesmo nome do arquivo e este deve ser em letras minúsculas.
    • Não use o comando package em sua solução.
    • Java tem um desempenho inferior em comparação a C/C++ para tarefas que envolvem muito processamento, mas facilita a manipulação de strings e objetos.
    • Fique atento ao uso de bibliotecas padrão para entrada e saída, como Scanner e BufferedReader, que são suficientes para a prova.

Python

  • Extensão de arquivo: .py
  • Funções de entrada/saída: input(), print(), read(), readline(), readlines()
  • Dicas
    • Python é uma das linguagens mais simples para escrever código, devido à sua sintaxe limpa e interpretada.
    • Porém, cuidado com o desempenho em problemas que exigem alto processamento, pois Python pode ser mais lento.
    • Utilize listas, dicionários e outras estruturas embutidas para maximizar a eficiência.
    • Use input() para entrada de dados e print() para saída. Lembre-se de que tudo deve ser tratado na entrada/saída padrão (teclado e tela).

JavaScript

  • Extensão de arquivo: .js
  • Funções de entrada/saída: scanf, printf
  • Dicas
    • Embora menos comum em competições de programação, JavaScript é permitido e pode ser uma opção interessante.
    • O ambiente de execução para JavaScript normalmente requer um pouco mais de atenção para garantir que o formato de entrada e saída esteja correto.

Dicas Importantes para a Prova

  • Leia as Instruções Cuidadosamente: Cada tarefa da OBI tem um formato específico de entrada e saída. O seu programa deve obedecer rigorosamente esse formato, pois a correção é automatizada. Evite mensagens desnecessárias como "Digite o valor de entrada", e certifique-se de seguir as especificações da tarefa.
  • Eficiência é Fundamental: As tarefas não só exigem soluções corretas, mas também eficientes. Se o seu programa demorar muito para processar, ele poderá falhar em testes maiores. Esteja atento à complexidade do seu algoritmo e escolha as melhores abordagens para otimizar seu código.
  • Testes Extensos: Sua solução será testada com diferentes entradas além dos exemplos fornecidos na tarefa. Por isso, é crucial que seu código esteja preparado para lidar com cenários variados e maiores do que os de exemplo.
  • Submissão e Avaliação: Você pode submeter até 50 soluções para cada tarefa. A pontuação final será baseada na melhor submissão. Use isso a seu favor para aprimorar sua solução gradualmente, tentando melhorias de desempenho ou corrigindo erros.
